## 🔧 How It Works

### Question Generation Process:
1. **Job Analysis:** Extract skills from job title and description
2. **Template Selection:** Filter templates by job category (e.g., AI jobs → AI templates only)
3. **Content Matching:** 
   - Priority 1: Job-specific skills
   - Priority 2: Role-appropriate content (STRICT - no fallback)
   - Skip question if no match found
4. **Difficulty Distribution:** 10 Easy + 10 Medium + 10 Hard = 30 questions
5. **Deduplication:** Ensure unique questions

### Strict Role Matching Examples:
- **AI Engineer job** → Gets only AI category content (TensorFlow, neural networks, model deployment)
- **Mobile Developer job** → Gets only Mobile category content (Flutter, push notifications, app crashes)
- **DevOps Engineer job** → Gets only DevOps/Cloud category content (Kubernetes, CI/CD, deployments)
- **Web Developer job** → Gets only Web category content (React, responsive design, browser issues)

**NO MORE MIXING!** Web content won't appear in AI jobs, and AI content won't appear in Web jobs.
